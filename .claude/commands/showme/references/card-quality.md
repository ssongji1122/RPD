# ShowMe Card Quality Criteria

## 품질 철학

> "학생이 이 카드 하나로 해당 개념을 **정확히 이해하고 즉시 실습**할 수 있어야 한다."

카드는 단순 정보 나열이 아니라, **인지 부하를 최소화한 학습 경험**이다.

## 품질 기준 체크리스트

### 1. 개념 정확성
- [ ] Blender 공식 문서와 용어/동작이 일치하는가?
- [ ] 비유(analogy)가 개념을 왜곡하지 않는가?
- [ ] 단축키가 현재 Blender 버전(5.0+)에서 유효한가?

### 2. 학습 효과
- [ ] 개념탭: 핵심 개념이 3개 이내 카드로 구조화되어 있는가?
- [ ] 비유가 대상 청중(비전공 초보자)에게 직관적인가?
- [ ] 퀴즈: 브리프의 학생 혼란 포인트를 반영하는가?
- [ ] 퀴즈 explanation이 "왜 틀렸는지"를 설명하는가?

### 3. 인터랙션 품질
- [ ] Canvas 인터랙션이 Blender의 실제 파라미터를 반영하는가?
- [ ] 슬라이더/체크박스 조작이 즉각적으로 시각 피드백을 주는가?
- [ ] 터치 이벤트가 정상 동작하는가?
- [ ] requestAnimationFrame으로 부드럽게 렌더링되는가?

### 4. 사용 맥락
- [ ] "언제 쓰나요" 탭이 구체적 워크플로우 시나리오를 제시하는가?
- [ ] 추천 조합(combo)이 실제 Blender 파이프라인에서 유효한가?
- [ ] "안 필요할 때" 가이드가 대안을 제시하는가?

### 5. 기술 완성도
- [ ] 모바일 반응형 (`@media max-width: 600px`)
- [ ] postMessage 전송 정상
- [ ] initQuiz 정상 작동
- [ ] doc-ref 링크 유효
- [ ] 4탭 구조 완전

## Good Example vs Bad Example

### Good: 비유 (analogy)
```
Mirror Modifier는 "거울 앞에 물체를 놓는 것"과 같습니다.
거울(축)을 기준으로 반대편에 똑같은 형태가 자동으로 생깁니다.
원본을 수정하면 반사된 쪽도 함께 바뀝니다.
```
- 직관적, 1:1 대응, 동작 원리까지 설명

### Bad: 비유 (analogy)
```
Mirror Modifier는 복사-붙여넣기 같은 기능입니다.
```
- 대칭 개념 누락, Ctrl+D(Duplicate)와 혼동 유발

### Good: 퀴즈 explanation
```
오답입니다. Solidify는 면에 두께를 추가하는 모디파이어입니다.
"벽"을 만드는 것이지, 면 자체를 이동시키는 것이 아닙니다.
Extrude와 혼동하기 쉬운데, Extrude는 면을 복제하며 이동시킵니다.
```
- 오답 이유 + 정답 설명 + 혼동 포인트 해소

### Bad: 퀴즈 explanation
```
정답은 B입니다.
```
- 학습 기회 없음

## Quality Gate 절차 (Step 3.5)

카드 생성 후, 배포 전 아래 검증을 수행한다:

1. **자가 검증**: 위 품질 기준 체크리스트를 순회하며 pass/fail 판정
2. **실패 항목 수정**: fail 항목이 있으면 해당 부분만 수정 후 재검증
3. **최대 2회 재시도**: 2회 수정 후에도 fail이면 실패 사유를 로그에 기록하고 진행
4. **통과 기준**: 전체 체크리스트 항목 중 "기술 완성도" 5개는 필수 통과, 나머지는 80% 이상 통과

```
quality_gate_result: PASS | FAIL(reason)
retry_count: 0 | 1 | 2
```
