## TTS 나레이션 스크립트

### [인트로 — 0:00]
이번 영상도 두 파트입니다.
전반부는 G·R·S Transform 실습, 후반부는 Apply Transform과 Origin.
전반부는 바로 따라해보고, 후반부는 문제 상황을 먼저 보여드릴게요.

### [선택 확인 — 0:30]
먼저 큐브를 **좌클릭** 으로 선택합니다.
선택된 오브젝트는 주황색 테두리가 생깁니다.
Blender에서 무언가 조작하려면 항상 먼저 선택이에요. 이 습관이 중요합니다.

### [G Grab — 1:00]
**G키** 를 누르면 오브젝트가 마우스에 붙어서 따라다닙니다.
마우스를 움직여서 원하는 위치로 이동한 뒤 **좌클릭** 으로 확정하면 됩니다.
잘못됐다면 **우클릭 또는 ESC** 로 취소할 수 있어요.

해보세요. G를 누르면 큐브가 붙어서 움직이죠? 클릭으로 확정합니다.

### [축 제한 — 2:00]
G만 누르면 모든 방향으로 자유롭게 이동해서 정확한 위치에 놓기가 어려울 수 있어요.
이럴 때 **축 제한** 을 씁니다.

- **G + X**: X축 방향으로만 이동 (좌우)
- **G + Y**: Y축 방향으로만 이동 (앞뒤)
- **G + Z**: Z축 방향으로만 이동 (위아래)

G를 누르고 바로 X를 눌러보세요.
이제 X축 방향으로만 움직이죠?

### [수치 입력 — 3:00]
더 정확하게 이동하려면 숫자를 직접 입력합니다.
**G + X + 2 + Enter** 를 해보세요.
X축으로 정확히 2 유닛 이동됩니다.
G → 축 → 숫자 → Enter. 이 순서를 기억해두세요.

### [R Rotate — 4:00]
다음은 **R키** 입니다. 회전이에요.
R을 누르면 오브젝트가 마우스 위치에 따라 회전합니다. 클릭으로 확정, 우클릭으로 취소.

축 제한도 동일하게 됩니다.
**R + Z + 45 + Enter** → Z축 기준 45도 회전.
R+X, R+Y도 같은 방식이에요.

### [S Scale — 5:00]
마지막으로 **S키** 는 크기 조절입니다.
S를 누르면 마우스를 가운데서 멀리 할수록 커지고, 가까이 할수록 작아져요.

- **S + 0.5 + Enter** → 절반 크기로 축소
- **S + X + 2 + Enter** → X방향으로만 2배 확대

특정 축으로만 스케일을 조정하면 오브젝트를 납작하게 만들거나 늘릴 수 있어요.
로봇 몸체 만들 때 많이 쓸 거예요.

### [종합 연습 — 6:00]
이제 직접 해볼게요.
**Shift + A** 로 Sphere와 Cylinder를 추가해서 오브젝트를 세 개 만들어보세요.
각각 선택하고 G·R·S와 축 제한을 써서 원하는 위치에 배치해봅니다.
잘못됐으면 언제든 **Ctrl + Z** 로 되돌릴 수 있어요.

### [Apply Transform 문제 시연 — 7:00]
자, 이제 후반부입니다.
지금 당장은 왜 필요한지 와닿지 않을 수 있어요.
하지만 이걸 모르고 모델링을 진행하면 나중에 반드시 문제가 생깁니다.
실제로 어떤 문제가 생기는지 먼저 보여드릴게요.

큐브를 하나 선택하고 **S키** 로 크기를 2배 정도 키워봅니다.
그다음 오른쪽 Properties Panel, Object Properties 탭을 보세요.
Scale 값이 (2, 2, 2) 로 되어 있죠?

이게 무슨 의미냐면, Blender 내부적으로는 이 오브젝트의 진짜 크기가 아직 (1, 1, 1) 이고
그냥 화면에 2배로 크게 그리고 있는 거예요.
겉으로 보기엔 똑같아 보이지만, 내부 데이터는 달라요.

### [문제 발생 — 7:30]
이 상태에서 오른쪽 Properties 모디파이어 탭, 렌치 모양 아이콘을 클릭합니다.
**Add Modifier → Subdivision Surface** 를 추가해볼게요.

어떻게 됩니까? 형태가 이상하게 비틀리거나 예상과 다르게 나타나죠?
Scale이 적용되지 않은 상태에서 모디파이어를 썼기 때문이에요.
이런 문제가 리깅, 물리 시뮬레이션, GLB 내보내기 할 때도 똑같이 생깁니다.

### [해결법 — 8:00]
해결 방법은 간단합니다.
모디파이어를 지우고, 스케일을 바꾼 뒤에는 **Ctrl + A** 를 눌러서
**Apply All Transforms** 를 선택하세요.

이러면 Blender가 "이 크기가 진짜 크기야" 라고 데이터를 업데이트합니다.
실행하고 나면 Scale 값이 (1, 1, 1) 로 돌아온 걸 확인할 수 있어요.
오브젝트 크기는 그대로인데 내부 수치가 정리된 겁니다.

습관으로 외워두세요: **모디파이어 추가 전, 리깅 시작 전, 내보내기 전 → 반드시 Ctrl+A!**

### [Origin — 9:00]
마지막으로 **Origin** 입니다.
뷰포트에서 오브젝트를 보면 **주황색 작은 점** 이 하나 보이죠?
이게 Origin이에요. 오브젝트의 기준점입니다.
이동, 회전, 스케일의 중심이 바로 이 점을 기준으로 이루어집니다.

Origin이 엉뚱한 곳에 있으면, 회전할 때 오브젝트가 이상하게 돌아가요.

### [Origin 설정 — 9:30]
오브젝트를 선택하고 **우클릭** 하면 컨텍스트 메뉴가 뜹니다.
여기서 **Set Origin** 을 선택하세요.

세 가지 옵션이 있어요:
- **Origin to Geometry**: Origin을 메시의 기하학적 중심으로 이동. 가장 많이 씁니다.
- **Origin to 3D Cursor**: 3D Cursor 위치로 Origin을 이동. 정밀 배치에 유용해요.
- **Origin to Center of Mass**: 물리적 질량 중심으로 이동. 시뮬레이션에서 씁니다.

일반적으로 **Origin to Geometry** 를 가장 많이 씁니다.
오브젝트 편집 후 Origin이 중심에서 벗어나 있다면 이걸로 정리하세요.

### [정리 — 10:30]
오늘의 핵심 세 가지.

첫 번째, **G·R·S** + 축 제한 + 수치 입력. 이 세 키가 Transform의 전부입니다.
두 번째, **Ctrl+A → Apply All Transforms**: 모디파이어 추가 전, 내보내기 전 반드시.
세 번째, **Set Origin to Geometry**: Origin을 항상 오브젝트 중심에 맞추기.

이 세 가지를 습관으로 만들어두면 Week 03 모델링이 훨씬 수월해집니다.
다음 수업에서는 Edit Mode에 들어가서 본격적인 모델링을 시작합니다.
