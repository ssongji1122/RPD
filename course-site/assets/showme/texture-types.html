Procedural 절차적 텍스처

수식(알고리즘)으로 만드는 텍스처. Noise, Voronoi, Musgrave 등이 있어요. 이미지 파일이 필요 없고, 해상도 제한이 없습니다.

  • 장점: 무한 해상도, 파일 용량 0, 파라미터 변경만으로 무한 변형
  • 한계: 구체적 디테일(글자, 로고) 표현 어려움
만화경을 생각해보세요 — 규칙에 따라 자동으로 패턴이 생기지만, "정확히 이 그림"을 보여줄 수는 없어요

Image 이미지 텍스처

사진/그림 파일을 불러와서 표면에 입히는 텍스처. 벽돌, 나무결, 녹, 페인트 벗겨짐 등 구체적 재질 표현에 강합니다.

  • 장점: 정확한 디테일 표현, PBR 맵 세트 활용 가능
  • 한계: 해상도 고정, 파일 용량, UV 매핑 필요 (Week 7)
벽지를 붙이는 것과 같아요 — 원하는 그림을 정확히 입힐 수 있지만, 크기와 위치를 맞춰야 해요 (그게 UV!)
핵심: 둘 다 쓰는 게 정답! Procedural로 자연스러운 바탕을 깔고, Image Texture로 구체적 디테일을 입히는 조합이 프로 워크플로우입니다.

각 버튼을 눌러 Procedural과 Image Texture의 차이를 확인하세요

🎲 Procedural을 쓸 때

  • 추상적/유기적 패턴 (대리석, 구름, 물결)
  • Roughness/Bump 같은 보조 맵
  • 파일 용량을 줄여야 할 때
  • 빠른 프로토타이핑

🖼️ Image를 쓸 때

  • 구체적 디테일 (벽돌, 타일, 나무 판자)
  • PBR 맵 세트 (Poly, Poly Haven 등에서 다운)
  • 사실적 재질이 필요할 때
  • 로고, 텍스트 등 정확한 이미지가 필요할 때

🔀 섞어 쓸 때 (프로 팁)

  • Image Texture로 기본 재질 + Noise로 얼룩 추가
  • Image Roughness 맵에 Noise를 Mix해서 변형
  • MixRGB 노드로 두 텍스처 합성

⚠️ 주의할 점

  • Image Texture는 반드시 UV 매핑 필요 (Week 7)
  • Normal Map은 Color Space를 Non-Color로!
  • 이미지 경로가 깨지면 분홍색 표시
