Transform Orientation은 G/R/S 변환 시 "어느 방향이 X·Y·Z인지" 결정하는 설정이에요. 같은 G+X라도 Global이냐 Local이냐에 따라 이동 방향이 완전히 달라집니다.

개념 Transform Orientation이란?

변환 축의 방향을 결정해요. 나침반처럼 "어느 방향이 앞(X)이고 위(Z)인지" 기준을 정하는 것이에요. 헤더 드롭다운 또는 ,(쉼표) 파이 메뉴로 전환합니다.

비행기 조종사는 비행기의 '앞'을 기준으로 좌우/위아래를 판단해요. Global은 지도, Local은 기체 기준이에요.

6가지 종류 전체 목록

  • Global — 월드 좌표축 기준 (항상 X=오른쪽, Z=위)
  • Local — 오브젝트 자신의 축 (회전해도 오브젝트 기준 유지)
  • Normal — 선택한 Face/Edge의 법선 방향 (Edit Mode)
  • Gimbal — Euler 회전 기반 짐벌 축
  • View — 현재 뷰포트 화면 기준
  • Cursor — 3D Cursor 방향 기준

핵심 비교 Global vs Local vs Normal

항목 Global Local
기준 월드 좌표 오브젝트 자체
오브젝트 회전 시 축 방향 불변 축도 함께 기울어짐
G+Z 결과 항상 월드 위 오브젝트의 '위'
Normal 용도 선택 Face의 수직 방향 — 경사면에서 Extrude 시

Global — 월드 기준

  • 오브젝트를 월드 좌표 기준으로 정렬/배치
  • 회전 없는 오브젝트에서 기본 이동
  • 씬 전체 구조물 배치 (바닥, 벽 등)

Local — 오브젝트 기준

  • 기울어진 오브젝트를 자신의 '앞뒤'로 이동
  • 회전된 오브젝트를 자기 방향으로 밀기
  • G+Z Z (두 번) = Local Z 축 이동

Normal — 법선 기준

  • Edit Mode에서 경사면을 수직 방향으로 Extrude
  • 구의 각 면을 방사형으로 밀기
  • 굴곡면에서 정확한 돌출 방향 제어

커스텀 Orientation

  • 사용자 정의 방향 저장 후 재사용
  • Edge 선택 후 헤더 + 버튼으로 저장
  • 비정규 각도 구조물 작업 시 유용
흔한 실수 G+X로 이동했는데 오브젝트가 대각선으로 움직임
→
원인 Local이 설정된 상태에서 오브젝트가 기울어져 있어서 Local X가 대각선 방향임
흔한 실수 경사면에서 Extrude했는데 면이 비뚤어지게 돌출됨
→
원인 Global 기준으로 Extrude하면 경사면의 법선 방향이 아닌 월드 Z로 돌출됨
해결 기울어진 오브젝트를 자기 앞뒤로 이동하고 싶을 때
→
해결책 Local로 전환 후 G+Y (또는 G+Y Y) 적용

자주 쓰는 조합

Local + G/R/S
기울어진 오브젝트를 자기 방향으로 자연스럽게 이동·회전·스케일
Normal + Extrude
Edit Mode에서 경사면의 법선 방향으로 면 돌출 — 구나 굴곡 표면 작업
축 두 번 = Local 축
G+X X처럼 축 키를 두 번 누르면 Local 기준 X 이동 (전환 없이 빠르게)
. (마침표) vs , (쉼표)
마침표(.)는 Pivot Point(기준점), 쉼표(,)는 Orientation(방향) — 혼동 주의
빠른 Local 전환: Orientation을 바꾸지 않고도 G → X X(두 번)처럼 축 키를 두 번 눌러서 임시 Local 모드로 이동할 수 있어요. 일회성 Local 이동에 매우 유용합니다.
