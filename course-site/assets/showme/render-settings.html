Cycles Ray Tracing 엔진

빛의 경로를 물리적으로 추적하여 사실적인 이미지를 만듭니다. 반사, 굴절, 전역 조명(GI), Caustics 등을 정확하게 시뮬레이션합니다. 느리지만 결과물이 사실적입니다.

Cycles = 사진 촬영 — 시간이 걸리지만 진짜처럼 보여요

EEVEE 실시간 래스터 엔진

게임 엔진과 같은 래스터화 방식으로 빠르게 렌더링합니다. Screen Space Reflections, AO 등 근사 기법을 사용합니다. 빠르지만 물리적 정확도는 Cycles에 못 미칩니다.

EEVEE = 게임 화면 — 즉시 보이지만 완벽하진 않아요

비교 주요 차이점

  • 속도: EEVEE >> Cycles (수십~수백 배)
  • 정확도: Cycles >> EEVEE
  • 반사/굴절: Cycles 정확 vs EEVEE 근사
  • GI: Cycles 자동 vs EEVEE 수동 설정

주요 단축키

F12 렌더링 시작
Ctrl + F12 애니메이션 렌더링
F11 마지막 렌더 결과 보기

슬라이더를 좌우로 드래그하여 Cycles(왼쪽)와 EEVEE(오른쪽) 렌더 결과를 비교하세요.

Cycles EEVEE

왼쪽: Cycles (사실적 반사/그림자) | 오른쪽: EEVEE (빠르지만 근사)

Cycles를 쓸 때

  • 최종 프레젠테이션 / 포트폴리오
  • 건축 비주얼라이제이션
  • 제품 광고 렌더
  • 정확한 유리/금속 반사가 필요할 때

EEVEE를 쓸 때

  • 작업 중 빠른 프리뷰
  • 애니메이션 초안 확인
  • 게임 에셋 미리보기
  • 시간이 부족한 프로젝트
팁: 작업 중에는 EEVEE로 빠르게 확인하고, 최종 렌더만 Cycles로 전환하는 워크플로우가 가장 효율적입니다. Render Properties 패널에서 엔진을 바꿀 수 있습니다.

렌더링 최적화 팁

Cycles + Denoising
적은 Sample로 렌더 후 Denoiser로 노이즈 제거. 렌더 시간을 크게 단축할 수 있습니다.
EEVEE + Light Probes
Irradiance Volume과 Reflection Cubemap으로 EEVEE의 GI와 반사 품질을 향상시킵니다.
