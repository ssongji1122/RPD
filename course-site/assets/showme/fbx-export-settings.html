FBX는 Blender 모델을 Mixamo·AccuRIG·Unity·Unreal로 보내는 표준 포맷이에요. 내보내기 직전 단계와 옵션을 정확히 맞춰야 외부 도구에서 크기·축이 틀어지지 않습니다.

FBX (Filmbox)

3D 데이터 교환 표준 포맷이에요. Mesh, Material, Armature, Animation을 하나의 파일에 묶어서 다른 소프트웨어로 옮길 수 있어요.

서로 다른 회사의 3D 도구가 통신할 수 있는 공용 컨테이너 — 박스에 메쉬·뼈·재질·애니메이션을 함께 담아 보내는 격이에요.

왜 표준인가

Blender → Mixamo / AccuRIG 자동 리깅, Blender → Unity / Unreal 게임 엔진 전송에서 가장 안정적으로 동작하는 포맷이라 사실상 표준으로 굳어졌어요.

  • Mixamo: FBX만 업로드 허용
  • Unity·Unreal: FBX 임포트가 1순위 워크플로

사전 준비 3가지

Export 전에 모델을 정리해 두지 않으면 받는 쪽에서 크기·뼈가 어긋나요. 다음 세 가지는 무조건 끝내고 나가세요.

  • Ctrl+J 로 모든 파츠를 단일 메쉬로 합치기
  • Ctrl+A > All Transforms 적용 (Scale 정상화)
  • Mixamo 업로드 시: 기존 Armature 제거 (자동 리깅이 새로 생성)

Export 핵심 옵션

File > Export > FBX (.fbx) 진입 후, 우측 패널에서 다음 값을 맞춰야 Mixamo·AccuRIG가 모델을 똑바로 인식해요.

  • Transform > Apply Scalings: All Local
  • Forward: -Z Forward
  • Up: Y Up
  • Geometry > Apply Modifiers 체크

흔한 실수

Ctrl+A > All Transforms를 안 하면 Blender상으로는 정상으로 보여도, Mixamo에서 캐릭터가 1cm로 작아지거나 100m로 거대해져요. Scale 1.0을 강제로 적용된 상태로 만드는 게 핵심.

"100% 확대해 둔 사진"과 "원본 크기 100% 사진"은 외형은 같아도 정보가 달라요. 외부 도구는 원본 크기를 기준 삼기 때문에 적용을 안 하면 비율이 깨져요.

축 방향 (Forward / Up)

각 소프트웨어마다 "앞쪽"과 "위쪽" 축이 달라요. Blender의 기본 축을 Mixamo가 기대하는 축으로 변환해서 내보내야 T-Pose가 똑바로 섭니다.

  • Mixamo / AccuRIG: -Z Forward, Y Up
  • 축이 틀리면 캐릭터가 누워서 임포트됨

Export 직전 단축키

Ctrl+J 선택된 오브젝트들을 활성 오브젝트로 합치기 (Join)
Ctrl+A Apply 메뉴 — All Transforms / Scale / Rotation 선택
N 사이드바 열어 현재 Scale·Rotation 수치 확인
A 전체 선택 — Join 전에 모든 파츠 잡을 때
Blender 공식 문서 — FBX I/O

Blender FBX Export 패널을 그대로 재현했어요. 각 옵션을 클릭하면 설명이 펼쳐지고, 상단 토글로 "올바른 설정"과 "잘못된 설정"을 비교할 수 있어요.

각 행을 클릭하면 옵션 의미와 주의점이 펼쳐져요. 모드 토글로 잘못된 설정 시 어떤 일이 벌어지는지도 확인해 보세요.

Mixamo 자동 리깅 업로드

  • T-Pose 자세로 셀프 정리 후 Export
  • 단일 메쉬(Ctrl+J)로 합쳐서 업로드
  • Ctrl+A > All Transforms 필수
  • 기존 Armature는 미리 삭제

AccuRIG (Reallusion) 자동 리깅

  • Mixamo와 동일한 사전 준비
  • T-Pose + 단일 메쉬 + Apply Transform
  • 축은 -Z Forward / Y Up 권장
  • 얼굴 디테일이 있으면 ARP 호환성 체크

Unity / Unreal 게임 엔진

  • Armature 포함, Animation 포함 옵션 켜기
  • Geometry > Apply Modifiers 체크
  • Path Mode: Copy + Embed Textures로 텍스처도 같이 전송
  • Unreal은 Skeleton 호환을 위해 별도 옵션 확인

다른 3D 소프트웨어 데이터 교환

  • Maya · 3ds Max · Cinema 4D와 Mesh 주고받기
  • Substance Painter로 텍스처링 보내기 전
  • Marvelous Designer 의상 시뮬레이션 송수신
Export 시 선택지
같은 FBX라도 무엇을 함께 보내느냐에 따라 옵션이 달라요.

• Mesh-only: 모양만. 정적 prop, 환경 오브젝트
• Mesh + Armature: T-Pose 캐릭터. Mixamo·AccuRIG 자동 리깅용
• Mesh + Armature + Animation: 모션까지. 게임 엔진 임포트용

애니메이션을 함께 내보낼 때는 Bake Animation 옵션을 켜서, 컨스트레인트나 드라이버로 만든 동작도 키프레임으로 굽혀줘야 외부 도구에서 정상 재생돼요. Material 텍스처는 Embed 안 하면 별도 이미지 파일로 같이 묶어 보내야 합니다.

Mixamo 업로드 워크플로

① 모델 정리
A로 전체 선택 → Ctrl+J로 단일 메쉬화. 기존 Armature 삭제.
② Transform Apply
Ctrl+A > All Transforms로 Scale·Rotation 강제 적용.
③ FBX Export
File > Export > FBX → Apply Scalings: All Local, -Z Forward, Y Up.
④ Mixamo 업로드
mixamo.com에 .fbx 드래그 → 관절 마커 배치 → 자동 리깅 완료.
