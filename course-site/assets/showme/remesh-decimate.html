Decimate 폴리곤 수 줄이기

메시의 형태를 유지하면서 폴리곤 수만 줄이는 Modifier입니다. 0.0~1.0 사이의 Ratio로 얼마나 줄일지 결정합니다.

  • 원본 토폴로지 구조 유지
  • Ratio 1.0 = 원본 / 0.1 = 90% 감소
  • 형태는 유지, 폴리 수만 감소
지도를 단순화할 때처럼 — 해안선 모양은 그대로지만 선의 꺾임 수가 줄어드는 것

Remesh 토폴로지 재구성

기존 메시 구조를 완전히 무시하고, 새로운 균일한 폴리곤 그리드로 처음부터 다시 만드는 Modifier입니다.

  • Voxel 모드: 3D 복셀 격자로 재구성
  • Blocks/Smooth/Sharp: 픽셀 방식 재구성
  • 토폴로지 구조 완전히 새로 생성
구겨진 종이를 펼친 뒤 새 종이에 다시 그리는 것 — 형태는 비슷하지만 선의 구성이 완전히 달라짐

Decimate 특징
Ratio를 낮출수록 삼각형이 커지고 수가 줄어듦. 형태는 유지되지만 기존 토폴로지 왜곡 발생 가능
Remesh 특징
Voxel Size가 클수록 격자가 거칠어짐. 기존 UV·버텍스 그룹은 모두 초기화되므로 주의
Decimate vs Remesh — 상황별 선택

📉 Decimate 쓸 때

  • AI 생성 메시 파일 크기 줄이기
  • 게임 엔진용 저폴리 LOD 생성
  • 렌더링 시 폴리 수 경량화
  • 형태 유지하면서 빠르게 단순화

🔲 Remesh 쓸 때

  • 스컬팅 전 메시 균일화
  • AI 생성 불균일 메시 정리
  • Boolean 연산 후 토폴로지 수정
  • 스캔 데이터 정리·재구성

Decimate 쓰지 말아야 할 때

  • 스컬팅 전 (균일한 분포 필요)
  • 애니메이션 리그가 있을 때
  • UV 유지가 중요할 때 (토폴로지 변경됨)

Remesh 쓰지 말아야 할 때

  • UV·버텍스 그룹 유지 필요할 때
  • 정밀한 하드서페이스 디테일 있을 때
  • 파일 용량만 줄이고 싶을 때 (→ Decimate)

추천 조합 워크플로우

AI 메시 정리 파이프라인
AI 생성 → Remesh(Voxel) → Sculpt 세부 수정 → Decimate 경량화 → 렌더
스컬팅 준비
Remesh(Smooth)로 균일화 → 브러시로 조각 → Decimate로 최종 출력 최적화
빠른 Boolean 수정
Boolean 연산 후 엉망인 토폴로지 → Remesh로 리셋 → 원하는 해상도로 재구성
게임 에셋 최적화
고해상도 메시 완성 → Decimate(Collapse) → Ratio 0.3~0.5로 LOD 생성
주의: Remesh를 적용하면 기존 UV 맵과 버텍스 그룹이 전부 사라집니다. Texture Painting이나 Weight Paint를 이미 했다면 Remesh 전에 반드시 백업하세요.
