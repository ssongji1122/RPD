Apply Transform이란? 필수

Ctrl+A로 오브젝트의 Transform 값을 "기본값"으로 초기화합니다. Scale이 (2, 1, 1)인 박스에 Apply Scale을 하면, 현재 크기가 그대로 유지되면서 Scale 값이 (1, 1, 1)로 리셋됩니다.

줄자의 눈금을 현재 위치에서 0으로 다시 세팅하는 것과 같아요

왜 Apply해야 하나요? 주의

Scale이 1이 아닌 상태에서 Modifier를 적용하면 예상치 못한 결과가 나옵니다.

  • Bevel이 한쪽으로 찌그러짐
  • Array가 비균일하게 복제됨
  • Subdivision Surface가 비대칭으로 부풀어 오름
  • 물리 시뮬레이션의 크기가 엉뚱해짐

주요 단축키

Ctrl + A Apply 메뉴 열기
Ctrl + A → S Apply Scale만 적용
Ctrl + A → R Apply Rotation만 적용
N N-패널에서 현재 Transform 값 확인

Scale이 적용되지 않은 박스 vs Apply 후 박스에 동일한 Bevel Modifier를 적용한 비교입니다.

좌: Scale(2,1,1) 미적용 / 우: Apply Scale 후 — 같은 Modifier인데 결과가 다릅니다.

반드시 Apply해야 할 때

  • Modifier 추가 전 (Bevel, Array 등)
  • 물리 시뮬레이션 적용 전
  • UV Unwrap 전
  • 다른 파일로 Export 할 때
  • Boolean 연산 전

Apply 안 해도 되는 경우

  • 단순 렌더링만 할 때
  • 아직 모델링 초기 단계
  • Linked/Instanced 오브젝트
팁: 모델링을 시작하기 전에 습관적으로 Ctrl+A → All Transforms를 적용하세요. N-패널에서 Scale이 (1, 1, 1), Rotation이 (0, 0, 0)인지 항상 확인하는 것이 좋습니다.

Apply와 관련된 흔한 실수

Bevel이 찌그러진다
비균일 Scale(예: 2,1,1) 상태에서 Bevel을 추가하면 한쪽만 넓게 깎입니다. Apply Scale로 해결됩니다.
Array 간격이 다르다
Scale이 1이 아닌 축 방향으로 Array하면 간격이 비균일해집니다. Apply 후 다시 설정하세요.
