# Week 08: 중간고사 - 중간 프로젝트 발표

## 학습 목표

- [ ] 모델링과 텍스처를 완성하여 하나의 결과물로 제출할 수 있다
- [ ] 자신의 작업 과정과 결과를 체계적으로 프레젠테이션할 수 있다
- [ ] 다른 학생의 작품에 건설적인 피어 리뷰를 제공할 수 있다

## 중간 프로젝트 개요

중간고사는 Week 01~07까지 배운 모든 기법을 종합하여 완성한 **캐릭터 로봇 모델링 + 텍스처** 결과물을 제출하고 발표하는 형식으로 진행됩니다.

- **배점:** 전체 성적의 **35%**
- **형식:** 결과물 제출 + 발표 (5분) + Q&A (2분)
- **제출처:** Discord #midterm-project 채널 + 수업시간 발표

## 중간 프로젝트 요구사항

### 모델링 (Week 03~04 기법 활용)

- 로봇 또는 캐릭터의 완성된 3D 모델
- Edit Mode 도구 활용: Extrude, Loop Cut, Inset, Bevel 등
- Modifier 활용: Subdivision Surface, Mirror, Solidify, Array, Boolean 등
- 깔끔한 Topology: 불필요한 Vertex/Edge/Face 없이 정리된 메쉬
- Apply Transform (Ctrl+A) 완료: Location, Rotation, Scale 모두 적용

### 텍스처 / 재질 (Week 06~07 기법 활용)

- Principled BSDF를 활용한 Material 설정
- 부위별 Material 할당 (금속, 플라스틱, 유리, 발광 등)
- UV Unwrap 수행 및 UV 전개도 정리
- 텍스처 적용 (아래 방법 중 하나 이상)
  - AI 텍스처 생성 (Meshy AI, 나노바나나 등)
  - Texture Painting
  - PBR 텍스처 라이브러리 활용 (Poly Haven 등)
- Pack Resources (File > External Data > Pack Resources) 완료

### 렌더 이미지

- **최소 3장** 이상의 렌더 이미지 제출
- 다양한 각도에서 촬영 (정면, 측면, 3/4 뷰, 클로즈업 등)
- 적절한 조명 설정 (기본 조명 또는 HDRI 활용)
- 해상도: 최소 1920x1080 이상 권장

### .blend 파일

- 완성된 .blend 파일 제출 필수
- 텍스처 Pack Resources 완료 상태
- 파일명 규칙: `week08_학번_이름.blend`

## 렌더링 기본 설정 가이드

발표용 렌더 이미지를 제작할 때 참고하세요.

### 빠른 렌더 설정 (Eevee)

1. Render Properties > Render Engine: **Eevee**
2. Sampling > Render: **64~128** (품질과 속도의 균형)
3. Output Properties > Resolution: **1920 x 1080**
4. Output Properties > File Format: **PNG**
5. World > Surface > Color: 단색 배경 또는 HDRI

### 고품질 렌더 설정 (Cycles)

1. Render Properties > Render Engine: **Cycles**
2. Sampling > Render: **128~256**
3. Denoising: **On** (OptiX 또는 OpenImage Denoise)
4. Output Properties > Resolution: **1920 x 1080**
5. Light Paths > Max Bounces: **6~8**

### 구도 팁

- **정면 (Front):** 캐릭터의 전체 디자인을 보여주는 기본 앵글
- **3/4 뷰:** 입체감이 가장 잘 드러나는 앵글 (가장 많이 사용)
- **클로즈업:** 얼굴이나 디테일 부분을 강조
- **로우 앵글:** 아래에서 위로 올려다보는 앵글, 위엄 있는 느낌
- **Full Body:** 전신이 보이는 앵글, 포즈와 비율 확인

## 💡 발표 팁

발표 전 아래 사항을 꼭 확인하세요. 작은 차이가 인상을 크게 바꿉니다.

### 렌더 이미지 품질 확인

- **밝기/대비 확인:** 렌더 이미지를 다른 모니터나 프로젝터에서 열어보세요. 내 화면에서 괜찮아 보여도 발표장 프로젝터에서는 어둡게 보일 수 있습니다. **밝기를 약간 높게** 설정하는 것이 안전합니다.
- **배경 색상:** 순수 검정(#000000) 배경은 프로젝터에서 모델과 구분이 안 될 수 있습니다. 약간의 그라데이션이나 어두운 회색(#1A1A1A)을 추천합니다.
- **이미지 포맷:** PNG는 무손실이라 품질이 좋지만, 파일 용량이 큼. JPG(품질 90% 이상)도 발표용으로 충분합니다.

### 발표 전 파일 백업

- 발표 당일 아침에 .blend 파일과 렌더 이미지를 **USB + 클라우드(Google Drive 등)** 두 곳에 백업
- 파일이 정상적으로 열리는지 **다른 컴퓨터에서 한 번 테스트**하는 것을 추천
- Pack Resources(File > External Data > Pack Resources)를 안 하면 다른 컴퓨터에서 텍스처가 깨짐

### 턴테이블 영상 준비 (가산점)

- 모델을 360도 회전하는 **턴테이블(Turntable) 영상**을 준비하면 가산점이 있습니다
- 제작 방법:
  1. Empty를 장면 중앙에 추가 (Shift+A > Empty > Plain Axes)
  2. 카메라를 Empty의 자식으로 설정 (카메라 선택 > Shift+클릭 Empty > Ctrl+P > Object)
  3. Empty에 Z축 Rotation Keyframe: Frame 1에서 0도, 마지막 프레임에서 360도
  4. Ctrl+F12로 Animation Render
- 5~10초 분량, 해상도 1280x720 이상 권장

## ⚠️ 흔한 감점 요인

매 학기 반복되는 감점 요인입니다. 제출 전 반드시 확인하세요.

### 1. Apply Transform 미적용

- **문제:** Ctrl+A(Apply All Transforms)를 안 하면 Scale이 1이 아닌 상태로 남아있어 이후 리깅/애니메이션에서 문제 발생
- **확인 방법:** Object 선택 > N 패널 > Transform에서 Scale이 (1, 1, 1)인지 확인
- **해결:** 모든 Object를 선택 > Ctrl+A > All Transforms

### 2. Material을 1가지만 사용

- **문제:** 로봇 전체에 하나의 Material만 적용하면 단조로워 보이고, 텍스처/재질 항목에서 감점
- **최소 기준:** 부위별로 **3종 이상**의 서로 다른 Material 할당 (예: 금속, 플라스틱, 발광)
- **팁:** Emission Material을 눈이나 버튼에 적용하면 시각적 효과가 크게 올라감

### 3. 렌더 해상도가 너무 낮음

- **문제:** 640x480이나 800x600 같은 낮은 해상도로 렌더하면 디테일이 보이지 않음
- **최소 기준:** 1920x1080 (Full HD)
- **팁:** Eevee로 빠르게 테스트 렌더 후, 최종 제출본은 Cycles로 렌더하면 품질 향상

### 4. 파일 용량 관리

- **문제:** 고해상도 텍스처를 사용하면 .blend 파일이 수백 MB가 될 수 있음
- **해결:** 텍스처 해상도를 2K~4K로 제한하고, 불필요한 텍스처는 제거
- **확인:** File > External Data > Report Missing Files로 누락된 텍스처 확인

### 5. Object 이름 미정리

- **문제:** "Cube.001", "Cylinder.003" 같은 기본 이름을 그대로 사용
- **해결:** 모든 Object에 의미 있는 이름을 부여 (예: "Head", "Body", "Arm_L", "Arm_R")
- **팁:** Outliner에서 이름을 더블클릭하면 바로 변경 가능
