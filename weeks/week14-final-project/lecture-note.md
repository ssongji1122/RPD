## 최종 프로젝트 요구사항

### 필수 제출물

#### 1. 완성된 로봇/캐릭터 3D 모델

- 모델링 완성 (Week 03~05 기법 활용)
- 텍스처/재질 완성 (Week 06~07 기법 활용)
- 리깅 완성 (Week 11~12 기법 활용)
- Armature와 Weight Painting이 적용된 포즈 가능한 모델

#### 2. 애니메이션 (5~10초)

- 자유 주제의 캐릭터 애니메이션
- Keyframe 기반 또는 Mixamo 애니메이션 활용
- 예시: 걷기, 인사, 춤, 전투 포즈, 감정 표현 등
- Timeline과 Graph Editor를 활용한 타이밍 조절

#### 3. 렌더링된 이미지 5장 이상

- 다양한 조명 환경 (3-Point Lighting, HDRI 등)
- 다양한 카메라 각도 (정면, 측면, 3/4 뷰, 클로즈업, 전신)
- Eevee 또는 Cycles 렌더러 사용
- 해상도: 최소 1920x1080

#### 4. 프로모션 영상 (선택, 가산점)

- 30초~1분 길이의 프로모션 영상
- AI 영상 도구 활용 가능 (Kling AI, Veo 등)
- BGM 추가 가능 (Suno AI, ElevenLabs Music 등)
- 렌더링된 애니메이션 + AI 생성 영상 편집

#### 5. .blend 파일 + 렌더 결과물

- Pack Resources 완료
- 파일명: `final_학번_이름.blend`
- 렌더 이미지 파일 (PNG 또는 JPG)
- 애니메이션 영상 파일 (MP4)

## 피드백 시간 운영

### 개별 1:1 피드백 (10분씩)

이번 주 수업에서는 교수자와 1:1로 피드백을 진행합니다.

- **순서:** 출석 번호순 또는 신청순
- **시간:** 1인당 10분 (전체 현황 확인 5분 + 피드백 5분)
- **준비물:** 현재까지 작업한 .blend 파일을 Blender로 열어둘 것

### 피드백 진행 방식

1. 현재 진행 상황 공유 (Viewport에서 모델 보여주기)
2. 체크리스트 기반 완성도 확인
3. 주요 이슈 식별 및 해결 방안 논의
4. 남은 1주간의 우선순위 작업 설정

### 우선순위 이슈 리스트

피드백 후 아래 형식으로 자신의 우선순위 이슈를 정리하세요:

```
[우선순위 1] (반드시 해결) 예: 오른팔 리깅 Weight Painting 수정
[우선순위 2] (가능하면 해결) 예: 걷기 애니메이션 타이밍 조절
[우선순위 3] (시간 여유 시) 예: 프로모션 영상 BGM 추가
```

## 기술적 문제 해결 FAQ

### 렌더가 너무 느릴 때

- **Eevee로 전환:** Cycles보다 훨씬 빠르며, Blender 5.0에서는 품질도 크게 향상됨
- **Render Samples 줄이기:** Render Properties > Sampling > Render 값을 128~256으로 설정
- **해상도 임시 축소:** 테스트 렌더 시 50%로 설정 후, 최종 렌더에서 100%로 복원
- **Light Bounces 줄이기:** Cycles > Light Paths > Max Bounces 값을 4~6으로 낮추기
- **불필요한 오브젝트 숨기기:** Outliner에서 렌더에 포함하지 않을 오브젝트의 카메라 아이콘 끄기

### 리깅이 깨질 때 (메쉬 변형 문제)

- **Weight Painting 확인:** Pose Mode에서 문제 부위 확인 후 Weight Paint Mode에서 수정
- **Automatic Weights 재시도:** Armature와 메쉬를 다시 Parent (Ctrl+P > Armature Deform > With Automatic Weights)
- **Vertex Group 정리:** 의도치 않은 Vertex Group이 영향을 주고 있는지 확인
- **Apply Transform 확인:** Armature와 메쉬 모두 Apply Transform이 되었는지 확인
- **Mixamo 재임포트:** 문제가 심한 경우 Mixamo에서 다시 리깅하여 FBX로 재임포트

### 텍스처가 깨질 때

- **UV 확인:** UV Editor에서 UV Island가 올바르게 펼쳐져 있는지 확인
- **Image Texture 경로 확인:** 외부 이미지 파일이 이동/삭제되지 않았는지 확인
- **Pack Resources:** File > External Data > Pack Resources로 텍스처를 .blend에 포함
- **UV 재작업:** 문제 부위의 Seam을 재설정 후 다시 Unwrap
- **텍스처 해상도:** 텍스처 이미지가 너무 작으면 픽셀이 보임 (최소 1024x1024 권장)

### 애니메이션이 부자연스러울 때

- **Graph Editor 활용:** Keyframe 사이의 Interpolation을 Bezier로 설정
- **Ease In/Out:** 동작의 시작과 끝에 감속 효과 추가
- **참고 영상:** YouTube에서 실제 로봇 또는 캐릭터 움직임 참고
- **12 Principles of Animation:** Squash & Stretch, Anticipation, Follow Through 등 참고
- **프레임 수:** 24fps 기준으로 동작 타이밍 설정

### Blender 크래시/파일 손상

- **자동 저장 복구:** File > Recover > Auto Save로 최근 자동 저장 파일 복구
- **Backup 파일:** .blend1 파일을 .blend로 이름 변경하여 열기
- **수시 저장 습관:** Ctrl+S를 자주 눌러 저장하고, 버전별로 파일을 따로 보관
- **Incremental Save:** File > Save As로 `final_v01.blend`, `final_v02.blend` 등 버전 관리

## ⚠️ 기말 프로젝트 흔한 감점 요인

> **매 학기 반복되는 감점 요인을 미리 확인하고 방지하세요!**

### 감점 1: 파일 정리 미흡 (텍스처 경로 깨짐)

- **상황:** 제출한 .blend 파일을 열면 텍스처가 분홍색(Missing Texture)으로 표시됨
- **원인:** 외부 텍스처 파일을 Pack하지 않아 경로가 깨짐
- **방지법:** 제출 전 반드시 `File > External Data > Pack Resources` 실행
- **확인법:** .blend 파일을 다른 폴더로 복사해서 열어보기 (텍스처가 정상인지 확인)

### 감점 2: 발표자료에 제작과정 미포함

- **상황:** 최종 결과물만 보여주고 제작 과정 설명이 없음
- **원인:** 평가 기준의 "발표" 항목에는 과정 설명이 포함됨
- **방지법:** 제작 과정 스크린샷을 최소 5장 이상 준비 (모델링 단계, 텍스처 적용, 리깅, 애니메이션, 렌더링)
- **팁:** 비포/애프터 비교 이미지가 성장을 보여주는 데 가장 효과적

### 감점 3: 렌더 퀄리티 부족

- **상황:** 렌더 이미지가 흐리거나 노이즈가 많거나 해상도가 낮음
- **원인:** 해상도가 1920x1080 미만이거나, Samples가 너무 낮거나, Denoiser 미사용
- **방지법:**
  - 최소 해상도: **1920x1080** (Full HD)
  - Cycles: 128 Samples 이상 + Denoising 활성화
  - Eevee: 64 Samples 이상 + AO/SSR 활성화
  - 렌더 전 Color Management > View Transform이 **AgX**인지 확인

### 감점 4: 포트폴리오 구성 미흡

- **상황:** 렌더 이미지가 모두 비슷한 각도이거나, 조명이 단조로움
- **원인:** 다양한 앵글/조명 시도 부족
- **방지법:**
  - 최소 5장의 렌더 이미지를 **서로 다른 각도와 조명**으로 준비
  - 권장 구성: 정면 3/4뷰 + 측면 + 클로즈업(얼굴/디테일) + 전신 + 분위기 연출 샷
  - 턴테이블 영상도 있으면 큰 가산점

### 감점 5: 파일명 규칙 미준수

- **상황:** 파일명이 규칙과 다르거나 한글이 포함되어 열리지 않음
- **방지법:** 파일명 규칙을 정확히 따르기: `final_학번_이름.blend`

> **💡 프로 팁: Pack External Data로 파일 안전하게 관리**
> - `File > External Data > Pack Resources`를 실행하면 모든 외부 텍스처, HDRI, 이미지가 .blend 파일 안에 포함됩니다.
> - 파일 크기가 커지지만, 다른 컴퓨터에서 열어도 텍스처가 깨지지 않습니다.
> - 제출 전 **반드시** 실행하세요. 이것만으로도 감점을 크게 줄일 수 있습니다.

> **💡 프로 팁: 제작과정 GIF로 인상적인 발표**
