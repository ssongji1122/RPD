# Week 1 기획 심화 (Design Doc)

> **목적:** curriculum.js의 콘텐츠가 아닌, 수업 기획 자체의 질을 검증하기 위한 문서.
> Content Lead + CPO가 Rubric 기반으로 주차별 기획을 심층 리뷰할 때 사용한다.

**주제:** 오리엔테이션 + AI 무드보드 (Mixboard)
**수업일:** 2026-03-04 (화) 예정
**강의실:** 인하대학교 (60주년관)
**Duration:** ~2시간
**Status:** done
**리뷰 완료일:** 2026-04-05

## 1. Student Journey (학생 여정)

### 시작 상태 (Week 1 진입 시점)
- Blender 경험 없음 (대부분 처음)
- "로봇프러덕트 디자인" 수업이 어떤 내용인지 모름
- 자신이 만들고 싶은 로봇/캐릭터 컨셉이 없는 상태

### 끝 상태 (Week 1 종료 시점)
- [ ] 수업 목표와 평가 기준을 이해한다
- [ ] Mixboard(또는 나노바나나)로 AI 이미지 생성 가능
- [ ] 자신만의 로봇/캐릭터 무드보드 1개 완성
- [ ] 컨셉 한줄 요약 가능 ("나는 ___을 만들 예정이에요")
- [ ] 본인 학생 페이지에 업로드 완료

### 핵심 전환점 ("아하!" 모먼트)
1. **AI 프롬프트가 구체적일수록 이미지가 달라질 때** — "cute robot" vs "small boxy robot with cat ears, LED eyes, matte finish" 결과 비교
2. **무드보드가 완성되는 순간** — 흩어진 이미지들이 하나의 컨셉으로 수렴되는 경험
3. **"내가 이걸 3D로 만들 수 있겠다"는 실감** — 무드보드를 보며 Week 15 완성작을 상상할 때

## 2. Stuck Map (예상 막힘 지점)

| Step | 예상 막힘 | 원인 | 대응 |
|------|----------|------|------|
| Mixboard 접속 | "사이트가 안 열려요" | Google 계정 미로그인, 지역 제한 | Google 계정 로그인 선행, VPN 비활성화 안내 |
| Mixboard 접속 | 한국 지역 접속 제한 발생 | Google Labs 지역 제한 | 대체 도구(나노바나나) 즉시 안내 |
| 프롬프트 입력 | 이미지가 원하는 것과 다름 | 너무 짧고 모호한 프롬프트 | 형용사 3개 이상 + 형태/색상/분위기 조합 예시 제공 |
| 무드보드 구성 | 이미지가 다 비슷하게 생김 | 같은 키워드 반복 사용 | 핵심 키워드만 고정하고 세부 사항을 바꾸도록 안내 |
| 컨셉 결정 | "뭘 만들지 모르겠어요" | 참고 레퍼런스 부족 | 팝마트, 키디랜드, 반려로봇 트렌드 이미지 3장 제시 |
| 컨셉 결정 | 컨셉이 너무 광범위 | "귀여운 로봇"처럼 방향성 없음 | 이름 + 성격 + 형태 + 색상 4개 항목 채우기 유도 |
| 학생 페이지 업로드 | 채널을 못 찾음 | 처음 오는 학생 | 화면 보여주며 #week01-assignment 위치 직접 안내 |
| 과제 내용 혼란 | "무드보드를 제출하는 건지 설치 스크린샷을 제출하는 건지" | curriculum.js vs assignment.md 불일치 → **P0 수정 필요** | 수업 중 구두로 명확히 안내 (무드보드 제출) |

## 3. 평가 정합성 (Assignment Alignment)

### 과제
> Mixboard 또는 나노바나나로 제작한 무드보드 이미지 2장 + 캐릭터/로봇 컨셉 한줄 설명

### 학습 목표 ↔ 수업 Step ↔ 과제 매핑

| 학습 목표 | 수업 Step | 과제 검증 |
|----------|-----------|----------|
| AI 도구로 이미지 생성 가능 | Mixboard 실습 (60분) | 무드보드 이미지 2장 이상 |
| 무드보드 구성 | 개인 실습 시간 | 이미지 배치 및 분위기 일관성 |
| 컨셉 한줄 정리 | 컨셉 잡기 세션 | 한줄 설명 포함 여부 |

### ⚠️ SSOT 불일치 발견 (P0)

| 파일 | 과제 제목 | 내용 |
|------|----------|------|
| `curriculum.js` | Blender 설치 확인 스크린샷 | Blender 실행 후 Welcome Screen 캡처 제출 |
| `assignment.md` | 무드보드 제작 | AI 도구로 무드보드 제작 + 컨셉 한줄 제출 |
| `slides.md` | 과제 안내 | 무드보드 이미지 2장 + 컨셉 한줄 설명 |
| `lecture-note.md` | 과제 | 무드보드 이미지 2장 + 컨셉 한줄 설명 |

**결론:** curriculum.js만 "Blender 설치 스크린샷"이고 나머지 3개 파일은 "무드보드"로 일치. curriculum.js 수정 필요.

### 채점 기준
- 창의성 (40%) — 독창적인 캐릭터/로봇 컨셉
- AI 도구 활용 (30%) — 프롬프트 작성 능력, 도구의 효과적 사용
- 컨셉 명확성 (30%) — 방향성이 명확하게 전달되는가

## 4. 의존성 체크

### 전주차 연결
- Week 1은 첫 주차 → 전제 없음 ✅
- Blender 미설치 상태가 정상 → lecture-note에 "Blender 설치는 Week 02에서" 명시됨 ✅

### 다음 주차 연결 (Week 2: Blender 인터페이스 + MCP)
- Week 1 무드보드 컨셉 → Week 2부터 시작하는 모델링 방향 기준점
- 미리 Blender 설치 권장 안내 필요 (Week 2 시작 전 집에서 설치) ⚠️ — 현재 lecture-note에 "Week 2에서 함께 진행" 안내는 있으나 사전 설치 독려 문구가 약함
- Week 5 AI 러프는 Week 1 무드보드 컨셉 이미지 기반으로 생성됨 → **Week 1 컨셉의 방향성이 전체 프로젝트의 출발점**

## 6. Rubric 자가 평가 (1-5점)

> Content Lead 리뷰 — 2026-04-05

| 차원 | 점수 | 근거 |
|------|------|------|
| 1. 학습 목표 구체성 | 3 | 무드보드 제작 목표는 명확. 그러나 "컨셉을 1차로 결정한다"가 측정 기준이 모호. 무드보드 이미지 수, 컨셉 키워드 항목 수로 구체화 필요. |
| 2. 전제 명시 | 4 | 첫 주차라 전제 조건이 거의 없음. AI 도구 접속 대안(나노바나나) 명시되어 있고, Blender 설치 미진행임을 lecture-note에 명시. 양호. |
| 3. Stuck Map 완성도 | 3 | Mixboard 지역 제한, 프롬프트 어려움, 컨셉 결정 어려움 등 주요 막힘 포인트 커버됨. 학생 페이지 제출 관련 혼란과 SSOT 불일치에서 오는 과제 혼란 항목 추가 필요. |
| 4. 평가 정합성 | 2 | **Major mismatch:** curriculum.js만 "Blender 설치 스크린샷"으로 과제 내용이 다름. 나머지 3개 파일은 "무드보드"로 일치. P0 수정 필요. |
