# 2026 Spring RPD Notion 재구성 청사진

기준일: 2026-03-07

이 문서는 기존 공개 페이지를 건드리지 않고, 새로운 노션 구조를 별도로 만드는 전제로 작성했다.

## 1. `복붙 가능한 형태`가 의미하는 것

여기서 말한 `복붙 가능한 형태`는 아래 4가지를 바로 사용할 수 있게 정리하는 것을 뜻한다.

1. 새로 만들 페이지 이름
2. 데이터베이스 이름과 속성명
3. 각 페이지에 들어갈 섹션 제목과 순서
4. 주차 페이지 / 학생 개인 페이지에 바로 넣을 본문 템플릿

단, 노션 데이터베이스 자체는 텍스트를 붙여넣는 것만으로 생성되지 않으므로, DB는 수동으로 한 번 만들어야 한다.

## 2. 현재 가능한 범위

현재 세션에서는 공개 노션 페이지 내용 확인은 가능했지만, 노션 워크스페이스를 직접 수정하는 MCP 도구는 이 대화에 연결되어 있지 않다.

- 확인된 상태: 로컬 Codex 설정에 Notion MCP 서버 항목은 존재함
- 제한: 이 세션에는 노션 페이지를 직접 생성/수정하는 전용 MCP 호출 도구가 노출되어 있지 않음
- 결론: 지금은 `설계안 + 템플릿 + 마이그레이션 순서`까지는 내가 확실히 정리할 수 있고, 실제 노션 생성은 사용자 측에서 새 페이지를 만든 뒤 적용해야 함

## 3. 새로 만들 최상위 구조

기존 페이지와 섞이지 않게 아래처럼 완전히 새 루트 페이지를 만든다.

```text
2026 Spring RPD v2
├─ 01 교수자 운영 허브
├─ 02 학생 대시보드
├─ 03 학생 개인 페이지
├─ 04 운영 아카이브
└─ 99 이전 페이지 링크
```

권장 공유 범위:

- `01 교수자 운영 허브`: 교수자만
- `02 학생 대시보드`: 학생 공개
- `03 학생 개인 페이지`: 학생별 개별 공유
- `04 운영 아카이브`: 교수자만
- `99 이전 페이지 링크`: 교수자만

## 4. 새 구조의 핵심 원칙

1. 학생 메인 화면에서는 `이번 주에 무엇을 해야 하는지`만 먼저 보여준다.
2. 교수자 원본은 항상 비공개 마스터 DB에서만 수정한다.
3. 학생에게 보여줄 내용은 `linked database view`로만 노출한다.
4. 교수자 내부 메모와 학생 공개 피드백은 같은 DB 안에서 숨기지 말고, 아예 분리한다.
5. 학생 개인 페이지는 공개 대시보드 하위에 두지 않는다.

## 5. 꼭 만들 데이터베이스

### 5.1 `주차 마스터`

용도: 강의 운영의 기준이 되는 핵심 DB

권장 속성:

| 속성명 | 유형 | 설명 |
|---|---|---|
| 제목 | Title | `Week 01: 오리엔테이션` 형식 |
| 주차코드 | Select | `Week 01` ~ `Week 15` |
| 수업일 | Date | 실제 수업일 |
| 상태 | Select | `초안`, `검토중`, `게시` |
| 학생공개 | Checkbox | 학생 대시보드 노출 여부 |
| 반 | Multi-select | `1반`, `2반`, `공통` |
| 과제마감 | Date | 해당 주차 과제 마감 |
| 이번주여부 | Checkbox | 학생 홈 최상단 노출용 |
| 요약 | Text | 카드에서 보여줄 짧은 소개 |
| 핵심도구 | Multi-select | `Blender`, `MCP`, `Mixamo`, `AI Texture` 등 |
| 업데이트일 | Last edited time | 자동 |

학생용 필터:

- `상태` = `게시`
- `학생공개` = 체크

홈 상단 `이번 주` 뷰 필터:

- `이번주여부` = 체크
- `상태` = `게시`
- `학생공개` = 체크

## 6. 학생 메인 대시보드 권장 구성

페이지명:

```text
02 학생 대시보드
```

학생이 첫 화면에서 보게 될 구조:

```text
2026 Spring RPD
강의 한 줄 소개

[상단 안내]
- 수업시간
- 교수 연락 방식
- 과제 제출 원칙

[1영역] 이번 주 수업
- 이번 주 목표
- 이번 주 실습
- 준비물
- 주차 페이지 바로가기

[2영역] 이번 주 과제
- 과제명
- 마감일
- 제출 형식
- 제출 가이드

[3영역] 공지사항
- 최근 3개

[4영역] 주차별 강의자료
- Week 카드 목록

[5영역] 도움말
- Q&A
- 자료방
- Blender 팁

[6영역] 작품 갤러리
- 우수작 / 공유작
```

### 6.1 학생 메인 대시보드에 바로 넣을 텍스트

아래는 페이지 상단 소개 문구 초안이다.

```text
2026 Spring RPD

Robotic Product Design 수업의 주차별 강의자료와 과제 안내를 확인하는 페이지입니다.
먼저 이번 주 수업과 과제를 확인하고, 필요한 경우 아래 주차별 자료와 참고 자료를 활용하세요.

수업 운영 원칙
- 매주 수업 자료와 과제는 이 페이지에 업데이트됩니다.
- 과제 제출은 본인 개인 페이지에서 진행합니다.
- 질문은 Q&A 또는 수업 시간에 바로 남겨주세요.
```

### 6.2 학생 메인 대시보드의 블록 배치 권장안

```text
H1. 2026 Spring RPD
Text. 강의 소개 2~3줄
Callout. 수업시간 / 연락 / 제출 원칙
Divider

H2. 이번 주 수업
Linked view: 주차 마스터
Filter: 이번주여부=체크, 상태=게시, 학생공개=체크
View: List 또는 Gallery

H2. 이번 주 과제
Linked view: 과제 마스터
Filter: 연결된 주차.이번주여부=체크, 학생공개=체크
View: List

H2. 공지사항
Linked view: 공지사항
Filter: 게시=체크
Sort: 날짜 내림차순
Limit: 최근 3개

H2. 주차별 강의자료
Linked view: 주차 마스터
Filter: 상태=게시, 학생공개=체크
View: Gallery

H2. Q&A
Linked view: Q&A
Filter: 공개=체크

H2. 자료방
Linked view: 자료방
Filter: 공개=체크

H2. Blender 팁
별도 팁 DB가 있으면 linked view, 아니면 자료방과 통합해도 됨

H2. 학생 작품 갤러리
공개 가능한 작품만 linked view
```

```text
이번 주에 배우는 핵심 개념과 실습 목표를 한 문장으로 적습니다.

1. 오늘의 목표
-

- 준비물:
- 설치 확인:
- 참고 링크:

- 개념 1
- 개념 2
- 개념 3

4. 실습 순서
1. 
2. 
3. 
4. 

5. 오늘의 결과물
- 학생이 최종적으로 만들어야 하는 결과를 짧게 설명

6. 과제 안내
- 과제명:
- 제출 형식:

- 링크 1

10. 업데이트 로그
- 2026-03-07: 초안 작성
```

### 7.1 주차 페이지를 더 읽기 쉽게 만드는 팁

- 긴 이론 설명은 접기 토글 안으로 넣는다.
- 학생이 해야 할 행동은 항상 페이지 상단 30% 안에 둔다.
- 이미지가 많은 실습은 `Step 1`, `Step 2`, `Step 3` 식으로 나눈다.
- 과제 안내는 페이지 하단이 아니라 중간 이상에 둔다.
- 색상 callout은 `중요`, `주의`, `팁` 정도만 제한적으로 쓴다.

## 8. 학생 개인 페이지 템플릿

학생 개인 페이지는 공개 대시보드와 분리된 별도 영역에서 생성한다.

루트 구조:

```text
03 학생 개인 페이지
├─ 1반
│  ├─ 김민서 (12243798)
│  ├─ 김지현 (12214306)
│  └─ ...
└─ 2반
   ├─ ...
```

공유 원칙:

- 각 학생 페이지는 `해당 학생 + 교수자`만 공유
- 반 페이지 자체는 학생 전체 공유하지 않는 것을 권장

학생 개인 페이지 템플릿:

```text
학생 이름 / 학번 / 반

이번 학기 목표
- 

이번 주 제출
- 이번 주에 제출할 과제와 링크를 여기에 정리합니다.

공개 피드백
- 교수자가 학생에게 공개해도 되는 피드백만 여기에 작성합니다.

주차별 제출 아카이브
Week 01
- 제출 이미지/링크
- 한 줄 회고

Week 02
- 제출 이미지/링크
- 한 줄 회고

Week 03
- 제출 이미지/링크
- 한 줄 회고

...

중간 프로젝트
- 작업 링크
- 피드백 반영 내용

최종 프로젝트
- 작업 링크
- 최종 설명
```

### 8.1 학생 개인 페이지에서 분리해야 하는 것

학생 페이지 안에 넣지 말아야 하는 정보:

- 내부 평가 메모
- 교수자 관찰 기록
- 다른 학생과 비교한 내용
- 공개 전 점수 논의

이 내용은 반드시 `교수자 내부 메모` DB에서 관리한다.

## 9. 교수자 운영 허브 권장 구성

페이지명:

```text
01 교수자 운영 허브
```

권장 배치:

```text
H1. 2026 Spring RPD 운영 허브

H2. 이번 주 작업
- 이번주여부가 체크된 주차 뷰
- 아직 게시 안 된 과제 뷰
- 확인 필요한 제출 뷰

H2. 마스터 데이터베이스
- 주차 마스터
- 과제 마스터
- 공지사항
- 자료방
- Q&A
- 학생 마스터
- 제출/공개피드백
- 교수자 내부 메모

H2. 반별 학생 관리
- 1반 학생 필터 뷰
- 2반 학생 필터 뷰

H2. 운영 메모
- 수업 후 수정할 점
- 다음 주 준비물
```

## 10. 기존 페이지에서 새 구조로 옮기는 순서

기존 내용이 섞이지 않게 하려면 아래 순서를 지킨다.

1. 새 루트 페이지 `2026 Spring RPD v2`를 만든다.
2. 그 안에 `01 교수자 운영 허브`, `02 학생 대시보드`, `03 학생 개인 페이지`를 먼저 만든다.
3. `주차 마스터` DB를 먼저 만들고 Week 01~15 행만 비워서 생성한다.
4. 기존 Week 페이지 내용을 한 주씩 열어서, 새 `주차 마스터`의 해당 페이지 본문으로 복사한다.
5. 과제 안내는 `과제 마스터`로 분리해서 각 주차와 relation으로 연결한다.
6. 학생 관련 기존 공개 페이지는 당장 삭제하지 말고, 새 개인 페이지 체계가 완성된 뒤 아카이브로 이동한다.
7. 학생 공개 시작 전, 메인 대시보드에는 학생 개인정보가 남아 있지 않은지 다시 확인한다.

## 11. 현재 페이지 기준으로 바로 정리 대상인 항목

현재 공개 구조에서 새 구조로 옮길 때 우선 재배치해야 하는 항목:

- `📖 강의자료` 영역: 유지하되 `주차 마스터` 기반으로 재구성
- `📋 과제 현황`: 학생용 과제 안내 DB로 재정의
- `📚 자료방`: 유지 가능
- `❓ Q&A`: 유지 가능
- `🔧 Blender 팁`: 유지 가능 또는 자료방 통합 가능
- `📝 과제 제출 현황`: 학생 공개 영역에서 제거, 교수자 운영 허브로 이동
- `📚 반별 포트폴리오`: 공개 메인에서 제거, 학생 개인 페이지 영역으로 이동
- 학생 실명 페이지 목록: 공개 페이지에서 제거

## 12. 바로 적용 가능한 최소 버전

일단 가장 적은 작업으로도 효과가 큰 버전은 아래다.

1. 새 페이지 `2026 Spring RPD v2` 생성
2. 학생 대시보드에는 `이번 주`, `공지`, `주차별 강의자료`, `Q&A`, `자료방`만 남김
3. `과제 제출 현황`과 `학생 목록`은 새 공개 페이지에서 제외
4. 학생 개인 페이지는 개별 공유로 분리
5. 교수자 기록은 내부 메모 DB로 분리

## 13. 다음 단계 제안

다음으로 내가 바로 해줄 수 있는 작업은 2가지다.

1. 학생 메인 대시보드용 문구와 섹션을 더 보기 좋게 다듬은 `실제 배치안` 작성
2. `주차 마스터`, `과제 마스터`, `학생 개인 페이지`를 위한 더 구체적인 `노션 생성 체크리스트` 작성

원하면 다음 문서에서는 아래 형식으로 더 구체화할 수 있다.

- 학생용 메인 대시보드 최종 문안
- 주차 페이지 템플릿 최종 문안
- 학생 개인 페이지 최종 문안
- DB 속성 생성 순서
