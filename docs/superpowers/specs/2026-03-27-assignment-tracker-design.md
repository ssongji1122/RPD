# 과제 제출 추적 + 학기 성적 관리 시스템

> Date: 2026-03-27
> Status: Approved
> Branch: claude/elastic-hawking

## 배경

학생들이 Notion 개인 페이지에 주차별 과제를 업로드하지만, 교수자가 25명(1반 14명 + 2반 11명)의 페이지를 일일이 열어 제출 여부를 확인할 장치가 없다.

## 목표

1. **자동 제출 감지** — 학생 페이지의 주차별 섹션을 스캔하여 제출/미제출 판별
2. **Notion 채점 DB** — 제출 현황을 Notion 데이터베이스로 관리
3. **학기 성적 종합** — 출석/과제/중간/기말 비율에 따른 자동 총점 산출
4. **course-site 대시보드** — admin에서 제출 현황을 한눈에 확인

## 성적 비율

| 항목 | 비율 |
|------|------|
| 출석 | 10% |
| 과제 | 20% |
| 중간고사 | 35% |
| 기말고사 | 35% |

## 구성요소

### 1. 과제 제출 DB (Notion)

**위치**: `03 학생 개인 페이지 운영` 페이지 하위에 새 데이터베이스 생성

**행**: 25명 × 15주 = 375행 (스크립트가 자동 생성)

**속성**:

| 속성명 | 타입 | 설명 |
|--------|------|------|
| 학생명 | Title | 이름 (학번) |
| 학번 | Text | 학번 |
| 분반 | Select | `1반` / `2반` |
| 주차 | Select | `Week 01` ~ `Week 15` |
| 제출 | Checkbox | 자동 스캔으로 채움 |
| 학생페이지 | URL | 해당 학생 Notion 페이지 링크 |

**Notion 뷰 3개**:
1. **분반별 현황** — 분반 그룹 + 주차 정렬 (테이블 뷰)
2. **이번 주 미제출** — 특정 주차 필터 + 제출=false
3. **전체 매트릭스** — Board 뷰, 주차별 그룹

### 2. 학기 성적 종합 DB (Notion)

**위치**: 과제 제출 DB와 같은 레벨

**행**: 25명 (학생당 1행)

**속성**:

| 속성명 | 타입 | 설명 |
|--------|------|------|
| 학생명 | Title | 이름 (학번) |
| 학번 | Text | 학번 |
| 분반 | Select | `1반` / `2반` |
| 출석 | Number | 수동 입력 (100점 만점 기준) |
| 과제제출수 | Rollup | 과제 DB에서 제출=true 개수 자동 집계 |
| 과제점수 | Formula | `과제제출수 / 15 * 100` (15주 기준 비율 환산) |
| 중간고사 | Number | 수동 입력 (100점 만점) |
| 기말고사 | Number | 수동 입력 (100점 만점) |
| 총점 | Formula | `출석 * 0.1 + 과제점수 * 0.2 + 중간고사 * 0.35 + 기말고사 * 0.35` |
| 등급 | Formula | A+ (95~) / A (90~) / B+ (85~) / B (80~) / C+ (75~) / C (70~) / D+ (65~) / D (60~) / F (~60) |

**Relation**: 과제 제출 DB와 학번 기준 relation → rollup으로 제출 수 집계 (학번은 unique key)

**알려진 Notion 페이지 ID**:
- 부모: `03 학생 개인 페이지 운영` → `31c54d6549718107a864f0dc8d16c45c`
- 1반: `31354d654971814e958ae58303ae8429`
- 2반: `31354d65497181359b62eea0cbe2b0a4`

### 3. 자동 스캔 스크립트

**파일**: `tools/scan_submissions.py`

**동작**:
1. 분반 페이지(1반, 2반)에서 학생 목록 가져오기
2. 각 학생 페이지의 children 블록 순회
3. `### Week XX` 헤딩을 기준으로 주차별 섹션 분리
4. 섹션 내 블록 분석:
   - `"여기에 과제를 업로드하세요"` placeholder만 있으면 → **미제출**
   - 이미지 블록(`image`), 파일 블록(`file`), 또는 placeholder가 아닌 텍스트 → **제출**
5. 결과를 과제 제출 DB에 업데이트 (기존 행의 `제출` 체크박스 토글)

**의존성**: `tools/notion_api.py` (기존 Notion API 래퍼 활용)

**실행**:
```bash
# 전체 스캔
python tools/scan_submissions.py

# 특정 주차만
python tools/scan_submissions.py --week 3

# 특정 분반만
python tools/scan_submissions.py --class 1
```

**스캔 판별 로직**:
```
for each student_page:
  for each week_heading (h3):
    blocks = children under this heading until next h3
    if blocks contain only:
      - empty blocks
      - quote blocks with text matching "여기에 과제를 업로드하세요" or "여기에 중간 프로젝트를 업로드하세요" or "여기에 최종 프로젝트를 업로드하세요"
    then: submitted = false
    else: submitted = true
```

### 4. DB 초기화 스크립트

**파일**: `tools/init_grading_db.py`

**동작**:
1. Notion에 과제 제출 DB 생성 (속성 정의 포함)
2. 1반/2반 학생 목록을 읽어 375행 자동 생성
3. Notion에 성적 종합 DB 생성
4. 25명 학생 행 생성 + relation 연결
5. 뷰 3개 생성

### 5. course-site 대시보드

**위치**: admin.html에 "과제 현황" 탭 추가

**UI**:
- **분반 선택**: 1반 / 2반 / 전체 토글
- **히트맵 테이블**: 행=학생, 열=주차 (Week 01~15)
  - 초록: 제출
  - 빨강: 미제출
  - 회색: 해당 주차 아직 안 됨
- **요약 카드**: 이번 주 제출률, 미제출자 명단
- **성적 종합 뷰**: 읽기 전용 테이블 (출석/과제/중간/기말/총점/등급)

**데이터**: Notion DB API로 실시간 조회 (admin-server.py에 엔드포인트 추가)

## 기술 결정

1. **Notion = source of truth**: 채점 데이터는 Notion DB에 원본 저장
2. **기존 학생 페이지 read-only**: 스캔만 하고 수정하지 않음
3. **stdlib only**: 기존 `notion_api.py` 패턴 유지, 외부 의존성 없음
4. **course-site는 읽기 전용**: admin 대시보드에서 Notion 데이터 조회만
5. **수동 실행 우선**: cron 자동화는 Phase 2에서 검토
