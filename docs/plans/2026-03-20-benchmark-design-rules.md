# Benchmark Design Rules — RPD Visual System

> 작성일: 2026-03-20
> 벤치마크: Linear (탐색), Brilliant (학습), Vercel/Geist (토큰)
> 상태: 승인됨

---

## 1. 벤치마크 사이트와 역할

| 벤치마크 | 역할 | 참조 범위 |
|---------|------|----------|
| **Linear** | 탐색 모드 (index, library, shortcuts) | 레일 비율, 카드 리스트, 다크 톤, 네비게이션 밀도 |
| **Brilliant** | 학습 모드 (deck.html) | 카드 학습 경로, 진도 표시, 색상 코딩, 여백 리듬 |
| **Vercel/Geist** | 디자인 토큰 시스템 | 스페이싱 스케일, 타이포 스케일, 3단계 배경, 시맨틱 컬러 |

### 참조 원칙
- 레이아웃 비율 → Linear
- 카드 밀도/간격 → Vercel
- 학습 여백/리듬 → Brilliant
- 토큰 체계 → Vercel/Geist
- 사이드바 구조 → Linear + Notion (보조)

---

## 2. 핵심 규칙: 탐색 vs 학습 모드

### 2.1 하나의 디자인, 두 개의 온도

같은 디자인 시스템을 쓰되 **배경 컬러 온도**로 모드를 구분한다.

```
탐색 모드 (index, library, shortcuts)
  → 쿨톤: 중성~차가운 그레이, 블루 악센트
  → "도구를 고르는" 환경

학습 모드 (deck.html)
  → 웜톤: 미세한 따뜻한 언더톤, 앰버/그린 터치
  → "콘텐츠에 집중하는" 환경
```

차이는 **의도적으로 미세하게** — 나란히 놓아야 구분되는 수준. 위화감 없이 컨텍스트 전환만 유도.

### 2.2 모드별 토큰 매핑

| 토큰 | 탐색 (쿨) | 학습 (웜) | 비고 |
|------|----------|----------|------|
| `--bg` | `#0A0A0A` | `#0D0B09` | 배경 — 웜은 red 방향 +2 |
| `--surface` | `#111111` | `#141210` | 카드/패널 면 |
| `--surface-strong` | `#1A1A1A` | `#1C1A17` | 강조 면 |
| `--text` | `#F5F5F7` | `#F0EDE8` | 본문 — 웜은 살짝 크림 |
| `--muted` | `#8E8E93` | `#9C9590` | 보조 텍스트 |
| `--key` | `#0A84FF` | `#0A84FF` | 메인 악센트 — 동일 |
| `--key-soft` | `#93C5FD` | `#93C5FD` | 소프트 악센트 — 동일 |

악센트 컬러(`--key`)는 모드에 상관없이 동일하게 유지 → 같은 사이트임을 인식.

### 2.3 페이지별 모드 분류

| 페이지 | 모드 | 이유 |
|--------|------|------|
| index.html | 탐색 (쿨) | 카드 훑어보기, 덱 필터링 |
| library.html | 탐색 (쿨) | 카드 브라우징, 검색 |
| shortcuts.html | 탐색 (쿨) | 단축키 DB 조회 |
| deck.html | 학습 (웜) | 순서 있는 학습, 진도 추적 |
| admin.html | 탐색 (쿨) | 관리 도구 |

## 3. 레이아웃 규칙

### 3.1 사이드 레일 (Linear 기준)

```
접힘: 56px  (뷰포트 3.9% at 1440px)
펼침: 220px (뷰포트 15.3% at 1440px)
모바일 (≤720px): 하단 탭바 56px
```

이 비율은 Linear(56/200), Supabase(220), Notion(224) 5개 프로덕션 시스템에서 검증됨.

**레일 아이템:**
- 높이: 40px (터치 타겟 충족)
- 아이콘: 20px, 레일 중앙 정렬
- 접힘 시: 아이콘만, 호버 시 플로팅 툴팁
- 펼침 시: 아이콘 + 라벨 인라인
- 섹션 간 갭: 8px (Notion 6px 참고하되 터치 고려 8px)
- 아이템 간 갭: 2px

**레일 섹션 헤더** (Notion 참조):
- 11px / uppercase / weight 600 / 배경 대비 40% 밝기
- "거의 안 보이는" 구조화 라벨

### 3.2 메인 콘텐츠 영역

```
max-width: 1180px
padding: 32px (탐색) / 48px (학습)
```

- 탐색 모드: 콘텐츠 밀도 높게, 패딩 32px
- 학습 모드: 여유롭게, 패딩 48px, 섹션 간 `12vh` 상단 여백 (Brilliant 참조)

### 3.3 그리드 시스템

**탐색 카드 그리드:**
```css
grid-template-columns: repeat(auto-fill, minmax(260px, 1fr));
gap: 16px;
```
- 1440px에서 3~4열
- 카드-갭 비율 약 15:1 (Vercel 기준)

**학습 덱 뷰:**
```css
max-width: 640px;
margin: 0 auto;
```
- 단일 컬럼, 중앙 정렬
- 카드 간 간격: 12px
- 읽기 집중 레이아웃

## 4. 스페이싱 스케일 (Vercel/Geist 기준)

```
4px / 8px / 12px / 16px / 24px / 32px / 48px / 64px
```

**이 값만 허용.** 10px, 14px, 18px, 20px, 28px 등 중간값 사용 금지.

| 용도 | 값 | 토큰 |
|------|-----|------|
| 아이콘-라벨 간격 | 8px | `--sp-2` |
| 카드 내부 패딩 | 16px | `--sp-4` |
| 카드 간 간격 | 16px | `--sp-4` |
| 섹션 간 간격 | 32px | `--sp-8` |
| 페이지 콘텐츠 패딩 | 32px (탐색) / 48px (학습) | `--sp-8` / `--sp-12` |
| 대형 섹션 구분 | 64px | `--sp-16` |

## 5. 타이포그래피 스케일

Vercel 6단계 기반 + 학습 모드 오프셋:

| 토큰 | 탐색 모드 | 학습 모드 | 용도 |
|------|----------|----------|------|
| `--text-nano` | 11px | 11px | 섹션 라벨 (uppercase) |
| `--text-xs` | 12px | 12px | 메타데이터, 칩, 뱃지 |
| `--text-sm` | 13px | 13px | 네비 라벨, 카드 보조 |
| `--text-base` | 14px | 16px | 본문, 카드 설명 |
| `--text-lg` | 16px | 20px | 카드 제목 |
| `--text-xl` | 20px | 24px | 섹션 제목 |
| `--text-2xl` | 24px | 32px | 페이지 제목 |

학습 모드에서 `--text-base` 이상이 +2~8px 커짐 → "읽기 모드" 시그널.

### font-weight 스케일

| 토큰 | 값 | 용도 |
|------|-----|------|
| `--fw-normal` | 400 | 본문 |
| `--fw-medium` | 500 | 네비 라벨, 캡션 |
| `--fw-semi` | 600 | 카드 제목, 섹션 타이틀 |
| `--fw-bold` | 700 | 페이지 제목, 강조 |

이 4단계만 허용. 620, 630, 650 등 사용 금지.

---

## 6. border-radius 스케일

| 토큰 | 값 | 용도 |
|------|-----|------|
| `--radius-sm` | 8px | 인풋, 작은 칩 |
| `--radius` | 12px | 카드, 버튼 |
| `--radius-md` | 16px | 서브 패널 |
| `--radius-lg` | 20px | 섹션 컨테이너 |
| `--radius-xl` | 24px | 사이드바, 검색바 |
| `--radius-pill` | 999px | 칩, 필터, 뱃지 |

기존 10px, 14px, 18px, 26px, 28px → 가장 가까운 스케일 값으로 통일.

## 7. 3단계 배경 레이어 시스템 (Vercel 기준)

```
Layer 0: --bg         → 가장 깊은 배경 (페이지)
Layer 1: --surface    → 카드/패널 면
Layer 2: --surface-strong → 강조 면 (hover, 선택됨)
```

각 레이어 간 밝기 차이: OKLCH 기준 10-12 단위.
- 너무 작으면 (8 미만) 구분 안 됨
- 너무 크면 (16 초과) 위화감

### 탐색 모드 레이어
```
Layer 0: #0A0A0A
Layer 1: #111111
Layer 2: #1A1A1A
```

### 학습 모드 레이어
```
Layer 0: #0D0B09
Layer 1: #141210
Layer 2: #1C1A17
```

## 8. 카드 디자인 규칙

### 8.1 탐색 카드 (index, library)

Linear 리스트 + Vercel 카드 밀도 참조:

```
높이: auto (콘텐츠 기반)
패딩: 16px
아이콘: 36px × 36px
기본 표시: 아이콘 + 제목 (1행)
덱 표시: 오른쪽 6px dot + 약칭
호버: border-color 변화만
```

- 12개 이상 보이면 설명 절대 기본 노출 안 함
- 설명은 호버 확장 또는 모달

### 8.2 학습 카드 (deck)

Brilliant 학습 타일 참조:

```
max-width: 640px
패딩: 24px
아이콘: 48px × 48px
기본 표시: 번호 + 아이콘 + 제목 + 진행 상태
진행 상태: 완료(초록) / 진행중(블루) / 대기(회색)
```

- 카드 클릭 → Show Me 카드 열기
- 교수자 메모가 있으면 카드 아래 작은 노트 표시

## 9. 덱 키컬러 규칙

6색 팔레트 (기존 유지):

| 이름 | 값 | `#111111` 대비 | 용도 |
|------|-----|--------------|------|
| blue | `#93C5FD` | 8.2:1 ✅ | 기본, 인하대 프리셋 |
| green | `#34D399` | 7.8:1 ✅ | 사용자 덱 |
| purple | `#A78BFA` | 5.4:1 ✅ | 사용자 덱 |
| amber | `#FBBF24` | 10.1:1 ✅ | 사용자 덱 |
| rose | `#FB7185` | 5.7:1 ✅ | 사용자 덱 |
| cyan | `#22D3EE` | 8.9:1 ✅ | 사용자 덱 |

**적용 범위 (엄격):**
- ✅ 덱 필터 칩의 아이콘 배경 + 보더
- ✅ 카드의 덱 표시 dot (6px)
- ✅ deck.html 헤더 악센트
- ❌ 본문 텍스트
- ❌ 버튼 배경
- ❌ 카드 전체 배경

## 10. 글래스모피즘 규칙 (Supabase 참조)

```css
/* 글래스 카드 */
background: rgba(22, 22, 22, 0.6);
backdrop-filter: blur(12px);
border: 1px solid rgba(255, 255, 255, 0.08);

/* 글래스 호버 */
background: rgba(22, 22, 22, 0.75);
border-color: rgba(255, 255, 255, 0.12);
```

- `blur(16px)` 초과 금지 — 저사양 GPU에서 렌더링 문제
- 글래스는 **탐색 모드에서만** 사용 (도구 환경 느낌)
- 학습 모드는 불투명 면 사용 (읽기 집중)

## 11. 금지 목록

1. 스케일에 없는 스페이싱 값 (10px, 14px, 18px, 20px, 28px)
2. 스케일에 없는 font-weight (620, 630, 640, 650)
3. 스케일에 없는 border-radius (10px, 14px, 18px, 26px, 28px)
4. 인라인 `<style>` 태그
5. 하드코딩된 `rgba()` — 토큰 사용
6. 12개 이상 카드 보일 때 기본 설명 노출
7. 한 페이지에 2개 이상 강조색 역할
8. 학습 모드에서 글래스모피즘
9. 덱 키컬러를 본문/버튼/카드 배경에 사용
10. `blur(16px)` 초과

## 12. 벤치마크 검증 방법

prototype 또는 실제 페이지 수정 후:

1. **Linear 비교**: 레일 비율, 네비 밀도, 카드 간격이 Linear과 유사한 리듬인지
2. **Brilliant 비교**: 학습 뷰 여백, 카드 크기, 진도 표시가 Brilliant의 호흡과 비슷한지
3. **Vercel 비교**: 토큰 값이 스케일에 맞는지, 3단계 배경이 적용됐는지

`/design-agent post-verify` 실행 시 이 3개 사이트 기준으로 비율 체크.
