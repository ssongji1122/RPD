# Course Site Style Guide

작성일: 2026-03-13  
대상: `/Users/ssongji/Developer/Workspace/RPD`

## 1. 목표

이 스타일 가이드는 `학생이 수업 내용을 빨리 이해하고`, `헷갈리지 않고`, `직접 따라할 수 있게` 만드는 것을 최우선으로 둔다.

참고 방향은 `Apple-like`한 인상이다.  
다만 특정 브랜드를 복제하는 것이 아니라 아래 특징만 가져온다.

1. 과한 장식보다 정보의 선명함
2. 큰 소리 대신 차분한 위계
3. 읽기 쉬운 타이포그래피
4. 적은 색, 명확한 강조
5. 화면을 열었을 때 즉시 구조가 보이는 레이아웃

---

## 2. 핵심 원칙

### 2.1 Low Noise

- 불필요하게 큰 타이틀, 두꺼운 강조, 과한 glow를 줄인다.
- 화면은 조용하지만, 중요한 행동은 분명해야 한다.

### 2.2 Read First

- 학생은 디자인보다 먼저 `무엇을 해야 하는지`를 읽어야 한다.
- 제목은 강하되 짧고, 설명은 짧되 정확해야 한다.

### 2.3 Action Oriented

- 페이지는 읽기 자료가 아니라 `행동 유도 인터페이스`여야 한다.
- 각 섹션은 설명만 두지 말고 `직접 할 것`, `완료 기준`, `다음 단계`를 포함한다.

### 2.4 Consistent Week Structure

- 15주 페이지는 모두 같은 구조를 유지한다.
- 학생이 주차가 바뀌어도 페이지 사용법을 다시 배울 필요가 없어야 한다.

## 3. 컬러 시스템

너무 튀는 네온 톤 대신, 중성 회색 기반 위에 절제된 블루 하나만 쓴다.

### 기본 컬러

- Background: `#0A0A0A`
- Background Soft: `#111214`
- Surface: `#17181A`
- Surface Strong: `#1C1D20`
- Surface Alt: `#141518`
- Border: `rgba(255, 255, 255, 0.08)`
- Border Strong: `rgba(255, 255, 255, 0.14)`
- Text Primary: `#F5F5F7`
- Text Secondary: `#A1A1AA`
- Text Strong Secondary: `#D4D4D8`
- Accent: `#0A84FF`
- Accent Strong: `#0071E3`
- Accent Soft: `#8EC5FF`

### 운용 원칙

1. 메인 강조색은 블루 하나만 사용한다.
2. 성공 상태 외에는 초록을 거의 쓰지 않는다.
3. 카드 구분은 색보다 `밝기 차`와 `보더`로 해결한다.
4. 배경은 완전 검정 하나로 밀지 말고 미세한 표면 차이를 둔다.

## 4. 타이포그래피

### 폰트 원칙

- 제목과 본문을 다른 캐릭터로 세게 나누지 않는다.
- 한 가족처럼 보이는 산세리프 계열로 통일한다.
- `강의 페이지`에서는 개성보다 가독성이 우선이다.

### 추천 스택

- Display / Heading:
  `-apple-system, BlinkMacSystemFont, "SF Pro Display", "SUIT", "Pretendard", "Apple SD Gothic Neo", sans-serif`
- Body:
  `-apple-system, BlinkMacSystemFont, "SF Pro Text", "SUIT", "Pretendard", "Apple SD Gothic Neo", sans-serif`

### 권장 사이즈

- H1: `40px ~ 56px`
- H2: `32px ~ 40px`
- H3: `26px ~ 32px`
- Section title: `24px ~ 30px`
- Body large: `18px`
- Body default: `16px`
- Caption / chip: `13px ~ 14px`

### 권장 weight

- H1/H2: `600 ~ 700`
- H3/section title: `600`
- Body: `400 ~ 500`
- Chips / labels: `500 ~ 600`

### 금지 규칙

1. H1을 64px 이상 키우지 않는다.
2. 제목에 과도한 초중량 weight를 남발하지 않는다.
3. 제목과 본문 모두 자간을 과하게 좁히지 않는다.

## 5. 레이아웃 시스템

### 기본 구조

모든 주차 페이지는 아래 구조를 기본으로 한다.

1. Week Hero
2. 오늘의 목표
3. 공식 개념/화면 이해
4. 직접 따라하기
5. 자주 하는 실수
6. 과제 / 제출
7. 참고 자료

### 폭과 여백

- 최대 콘텐츠 폭: `1120px ~ 1200px`
- 카드 간격: `16px`
- 섹션 간격: `80px ~ 120px`
- 내부 패딩: `20px ~ 28px`

### 히어로 규칙

- 히어로는 한 화면을 완전히 먹어도 되지만, 메시지는 짧게 유지한다.
- 큰 제목 아래에는 반드시 `바로 행동할 수 있는 요약`이 있어야 한다.
- 히어로만 화려하고 아래가 약한 구조는 피한다.

## 6. 컴포넌트 규칙

### 6.1 Step Card

- 가장 중요한 컴포넌트
- 반드시 아래 4개를 포함한다.
  - 무엇을 할지
  - 왜 하는지
  - 완료 기준
  - 직접 할 체크리스트

### 6.2 Video Card

- 짧은 시연 클립은 10초~60초 사이가 좋다.
- 긴 영상을 그대로 넣지 말고 핵심 동작만 짧게 보여준다.
- 제목은 `무엇을 보게 되는지` 중심으로 쓴다.

### 6.3 Starter File Card

- 각 단계에 대응하는 `.blend` 파일을 바로 내려받게 한다.
- 파일명은 수업 흐름 기준으로 명확해야 한다.
- 예: `week02_navigation_practice.blend`

### 6.4 Official Docs Card

- 학생이 검색으로 헤매지 않게 공식 문서 링크를 선별해서 제공한다.
- 가능하면 문서 이미지 미리보기를 같이 둔다.
- 공식 문서는 “보충 설명” 역할이어야지, 메인 설명을 대신하면 안 된다.

### 6.5 Progress UI

- 진도는 단순하고 조용해야 한다.
- 몇 개를 끝냈는지 명확히 보여주되, 과한 게임화는 피한다.

## 7. 콘텐츠 작성 규칙

### 제목

- 짧게
- 추상적이지 않게
- 행동 또는 인지 목표가 느껴지게

좋은 예:

- `프리퍼런스 세팅`
- `화면 움직여 보기`
- `첫 모델 만들어보기`

나쁜 예:

- `창의적인 모델링의 시작`
- `무한한 가능성을 열어보자`

### 본문

- 문장은 짧게
- 한 문단은 2~3문장 이내
- 한 카드 안에는 핵심 1개만

### 체크리스트

- 체크 한 번이 실제 행동 하나와 연결돼야 한다.
- `이해했다`보다 `실행했다`를 기준으로 쓴다.

## 8. 15주 운영 구조 제안

15주차까지 갈 계획이라면 `단일 HTML`보다 `실제 웹사이트`가 더 적합하다.

### 추천 정보 구조

1. 메인 페이지
2. 주차 목록 페이지
3. Week 01 ~ Week 15 페이지
4. 공통 자료실
5. 과제 / 제출 안내 페이지
6. 자주 하는 질문 페이지

### 공통 주차 템플릿

모든 주차는 같은 레이아웃 시스템을 공유하고, 내용만 바꾼다.

### 왜 웹사이트가 더 좋은가

1. 15주를 한 HTML에 넣으면 탐색과 유지보수가 급격히 나빠진다.
2. 공통 헤더, 진도 UI, 카드 스타일을 재사용할 수 있다.
3. 주차별 미디어와 파일 관리를 분리할 수 있다.
4. 나중에 검색, 태그, 과제 링크, 학생용 북마크 기능을 붙이기 쉽다.

## 9. 기술 추천

현재 목적에는 `작은 수업 사이트`가 가장 적합하다.

### 1안. Astro + MDX

적합한 경우:

- 콘텐츠 중심
- 주차 페이지가 많음
- 정적 사이트로 충분
- 유지보수 단순성이 중요

장점:

- 빠름
- 구조가 단순함
- Markdown/MDX 기반으로 주차 콘텐츠 관리가 쉬움

### 2안. Next.js

적합한 경우:

- 나중에 로그인
- 제출 상태
- 서버 기능
- 학생별 저장 기능

장점:

- 확장성이 큼
- 향후 기능 추가가 쉬움

### 현재 내 추천

지금 단계에서는 `Astro 또는 정적 사이트 구조`가 가장 좋다.

이유:

- 지금 필요한 건 앱 기능보다 `좋은 수업 페이지 시스템`이다.
- 콘텐츠와 자산이 많은 구조에서 단순하고 빠르게 운영하기 좋다.
- 나중에 필요하면 Next.js로 옮길 수도 있다.

## 11. 결론

이 수업은 `예쁜 소개 페이지`보다 `학생이 따라가기 쉬운 수업 인터페이스`가 더 중요하다.

그래서 방향은 아래가 맞다.

1. 과한 디자인을 줄이고 차분한 위계로 간다.
2. 각 주차 페이지는 같은 구조로 반복한다.
3. 단일 HTML이 아니라 `작은 수업 웹사이트`로 가는 것이 맞다.
4. 노션은 정리용, 웹사이트는 수업용으로 역할을 나눈다.
