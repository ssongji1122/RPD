# Pilot Runbook

## Scope

대학 또는 학원 1곳에서 4주 파일럿을 운영한다고 가정합니다.

## Weekly Rhythm

월요일
- 이번 주차 콘텐츠 수정
- `content_pipeline.py check/build` 실행
- 공개 사이트 검수

화요일
- 교수자 리허설
- Show Me 카드와 과제 흐름 점검

수업일
- 운영 이슈 기록
- 막히는 개념/도구/과제 제출 패턴 메모

주말
- KPI 집계
- 다음 주차 보완 포인트 정리

## KPI Collection

- 주차 활성 학습자
- step 완료율
- 과제 제출율
- 콘텐츠 수정 반영 시간
- 영상 제작 시간

## Pilot Deliverables

- 파일럿 종료 요약 1부
- 교수자 인터뷰 요약 1부
- 반복적으로 막히는 Show Me 카드/실습 단계 목록
- 다음 학기용 개선 backlog

## Exit Criteria

- 교수자가 수정 흐름을 독립적으로 수행한다.
- 공개 사이트에 개인정보/시크릿 노출이 없다.
- 콘텐츠 수정이 같은 날 반영된다.
- Show Me 카드와 과제 흐름이 최소 1회 이상 실제 수업에서 사용된다.
